──────────────────────────────────────────────────────────────────────
COMO ABRIR
──────────────────────────────────────────────────────────────────────
1. Abra a pasta do projeto no Unity Hub (2022.3.40f1).
2. O Unity vai importar e gerar os .meta dos scripts novos automaticamente
   (a arquitetura é toda por código, então não há referência de cena/GUID
   para quebrar).
3. IMPORTANTE: se algo não refletir, o Unity pode estar usando o último
   build válido. Sempre confira o Console: se houver erro VERMELHO, ele
   ignora a mudança. Corrija o erro e rode de novo.

──────────────────────────────────────────────────────────────────────
ITENS QUE DEPENDEM DO EDITOR / DE PACOTES (não dá 100% só por código)
──────────────────────────────────────────────────────────────────────
Estes precisam de ação no Editor ou de instalar um pacote — deixei o
equivalente possível por código e aponto o caminho:

• CINEMACHINE: a câmera atual (CameraController) já faz follow suave, shake,
  zoom e limites. Para usar o Cinemachine de verdade: Window > Package Manager
  > instalar "Cinemachine" e trocar por uma Virtual Camera.

• BLOOM / MOTION BLUR / HDR PÓS-PROCESSAMENTO: o Built-in RP não tem isso
  sem o pacote "Post Processing" (Package Manager). Deixei HDR ligado na
  câmera + brilho por materiais emissivos e muitas luzes. Para bloom/motion
  blur: instale "Post Processing" e adicione um Post-process Volume + Layer.

• ADDRESSABLES: precisam do pacote "Addressables" e de marcar assets no
  Editor. Hoje o carregamento usa Resources + cache (rápido e suficiente).

• OCCLUSION CULLING / SPRITE ATLAS: são baking/configuração no Editor
  (Window > Rendering > Occlusion Culling; e criar um Sprite Atlas).
  Não são código; o Object Pooling e o cache já seguram o FPS.

Qualquer um destes eu integro se você instalar o pacote correspondente —
é só avisar.
